# 📒 额度小日记 · usage-diary

> 一个用来缓解 Claude 额度焦虑的小手账。

---

## 这是什么

这是一个**纯前端的小工具**，用来手动记录每次你看到 Claude 显示的"5 小时窗口"和"Weekly" 百分比，然后把这些数字可视化成曲线、汇率、预测。

做这个的根本动机不是"生产力"，是**缓解焦虑**——把一个让人心跳加快的数字变成一个可以"记一下"的动作、一段可视化的轨迹、一个温和的预测。焦虑被数据接住了，就没那么尖锐了。

它是给我自己做的。只有我一个人用。

---

## 功能

- **记一下**：用 +/− 步进按钮把两个百分比调到和 Claude 一致，一键保存
- **状态卡**：5h 窗口剩余时间、Weekly 本周进度、理想速率对比
- **汇率卡**：自动计算 "1% 5h ≈ 多少% Weekly"，区分高峰 / 非高峰时段
- **周曲线**：本周消耗的时间轴 + 按当前速率预测到周末会用到多少
- **备注**：每一笔记录可以挂一段"这一刻在干嘛"的手写体备注
- **云同步（可选）**：通过 [jsonbin.io](https://jsonbin.io) 在手机和 PC 之间同步所有数据
- **本地备份**：随时导出 JSON 到本地文件，也可以从文件导入恢复
- **深色模式**：跟随系统，也可以手动切换

## 怎么用

### 在线版本
部署在 GitHub Pages 上：打开对应的网址就能用。数据会存在浏览器的 localStorage 里。

### 本地运行
三个文件（`index.html`、`app.js`、`style.css`）是完全自包含的，没有任何构建步骤、没有依赖、不需要安装任何东西。

- **最简单的方式**：双击 `index.html`，用浏览器直接打开
- **推荐方式**（避免手机 Chrome 缓存问题）：用 Python 跑 `dev-server.py`

```
python dev-server.py
```

它会起一个 `localhost:8000` 的 HTTP 服务，带上 no-cache 头，改完代码刷新就能立刻看到。

---

## 数据存在哪里

**默认情况下，所有数据都只存在你的浏览器的 localStorage 里**——只有你这个浏览器、这台设备能看到。清浏览器数据会清掉它们，所以记得偶尔导出一份 JSON 存到本地或 U 盘。

**如果开了云同步**，数据会额外推一份到你自己的 jsonbin.io 账号下的一个存储桶里。Master Key 只存在你自己的浏览器里，**代码本身不包含任何密钥**。

## 技术栈

- 原生 HTML + CSS + JavaScript
- 图表用 [Chart.js](https://www.chartjs.org/)（通过 CDN 引入）
- 手写体字体用 [LXGW WenKai](https://github.com/lxgw/LxgwWenKai)（通过 CDN）
- 无 npm、无 webpack、无 React、无任何构建工具——**一个能打开浏览器就能跑**

## 设计原则

这个 App 不是照着"功能清单"做出来的，是照着**一些感受**做出来的。

1. **第一屏锁死** —— 打开就能记一笔，不用滚动、不用找按钮、不用想
2. **慢动画** —— 所有过渡都是缓慢、柔和的曲线，让人看着放松而不是被催
3. **治愈色调** —— 暖咖啡棕 + 蜂蜜黄 + 鼠尾草绿，避开所有让人紧张的高饱和红色
4. **零防御性 UX** —— 不弹"你确定吗？"不加"请先完成……"，相信用户知道自己在做什么
5. **多余空间要被合理吸收** —— 任何空白都应该有它存在的理由，不是 flex 没对齐遗留的废墟
